1. Introducción a los formularios

Los formularios permiten desarrollar aplicaciones web, de tal forma que se convierten en la única herramienta que permite recibir la información directamente del usuario final estructurada y según las necesidades del diseñador.

La primera preocupación de los desarrolladores de aplicaciones web es recuperar datos válidos, como, por ejemplo, un código postal o una dirección de email.

Con Html5 es posible validar dichos datos incluso cuando el usuario ha desactivado el código JavaScript del navegador.

El propósito último de los formularios es el procesamiento automático de los datos, siendo posible su gestión en lado del servidor mediante lenguajes de programación de gestión de bases de datos como, por ejemplo, PHP o MySQL.

2. Declarar el formulario

Para declarar el formulario se utiliza la etiqueta <form> que contiene todos los elementos del formulario.

Por sí misma, no muestra nada en el navegador.

Es compatible con todos los navegadores.

Debe cerrarse obligatoriamente. En caso contrario, el navegador no mostrará ningún campo del formulario.

Los atributos de <form> son:
name que permite asignar un nombre al formulario.
action que, cuando el usuario le da al botón de enviar, le dice la navegador la acción que debe llevar a cabo. Esta acción puede ser:
- Llamada a un programa de procesamiento de datos alojado en el servidor. Por ejemplo: action="http://www.servidor/procesamiento.php"
- Llamada a una dirección de e-mail para simplemente recuperar dichos datos utilizando el protocolo mailto. Por ejemplo: action="mailto:mi_correo@servidor.es"
- Llamada al procesamiento de los datos del formulario de forma interna (del lado cliente) mediante JavaScript, en cuyo caso el atributo action está vacío. Por ejemplo: action=""
OJO: la presencia del atributo action es obligatoria.
enctype especifica en que formato informático (mime type) se transmitirán los datos del formulario. Puede ser:
- Por defecto, application/x-www-form-urlencoded
- Para enviar archivos, este campo debe valer multipart/form-data
- Para el envío de datos a una dirección de correo electrónico mediante el protocolo mailto, el valor será text/plain
method indica cómo se va a realizar la transmisión de datos de un formulario. Puede ser:
- GET que realiza la transferencia de los datos en caracteres ASCII, pero dichos datos no pueden exceder los 100 caracteres.
- POST que gestiona los datos en caracteres no ASCII y tiene una capacidad de caracteres ilimitada. En la práctica, se ha impuesto por motivos de facilidad y eficacia.

OJO: cuando los datos del formulario se procesan de forma interna (es decir, cuando action="" los atributos method y enctype son inútiles por que no se hace llamada al servidor.

10. Botón de acción

La etiqueta <button> permite ejecutar, al hacer clic sobre el botón, una acción específica definida por el diseñador del sitio web. Generalmente, se ejecuta algún código de JS, aunque ofrece la opción de ser utilizada para enviar o restablecer el formulario, es decir, submit y reset.

Compatible con todos los navegadores, tiene etiqueta de cierre y su contenido se declara entre ambas etiquetas y no con el atributo value.

Sus atributos son:
name
type que admite tres argumentos:
- button utilizado para iniciar un script y valor por defecto en la mayoría de navegadores,
- submit para enviar el formulario, y
- reset para reiniciar el formulario.
En cualquier caso, se recomienda siempre especificar el valor de este atributo.
disabled que permite desactivar el botón cuando se carga la página. En este caso, aparece sombreado en gris y se habilita (enabled) mediante un script que se ejecuta tras una acción del usuario, por ejemplo, después de aceptar las condiciones de uso.

15. Etiquetado de los campos del formulario

La etiqueta <label> permite asociar el título a un campo de formulario concreto. De esta forma, se mejora la ergonomía, pues permite activar un elemento pinchando sobre el botó o sobre el título indistintamente. Pero también se mejora la accesibilidad a las personas invidentes.

Los pasos a seguir son:
1. Primero, el texto asignado a un elemento de formulario debe situarse entre las etiquetas <label>, es decir
<label>nombre del elemento</label>
<input type="text">
2. En segundo lugar, hay que asociar la etiqueta label al control del formulario correspondiente. Para ello, el elemento del formulario debe definirse mediante un identificador de tipo id. Es decir,
<label>nombre del elemento</label>
<input type="text" id="f1">
3.Por último, dentro de la etiqueta label, se utiliza el atributo for para asociar la etiqueta al campo identificado anteriormente. Es decir,
<label for="f1">nombre del elemento</label>
<input type="text" id="f1">

21. Campo de texto de búsqueda

Para ofrecer al usuario la posibilidad de buscar contenido concreto de nuestro sitio web, Html5 proporciona la etiqueta <input type="search"> que crea un campo de texto para ello.

Esta etiqueta no está implementada en todos los navegadores y precisa de un script posterior que demande en el servidor la búsqueda solicitada por el usuario. El atributo value contiene el DOMString que representa el valor contenido en el campo de búsqueda. Puedes recuperarlo usando la propiedad HTMLInputElement.value en JavaScript.
